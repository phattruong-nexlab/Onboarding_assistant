# Skill: init-project

## Trigger
`/init-project` — chạy một lần duy nhất khi apply SDD package vào project.
Cũng chạy khi Claude mới join một codebase chưa có docs.

## Mục đích
Scan toàn bộ codebase hiện có → tự động generate:
- `docs/ARCHITECTURE.md` — kiến trúc thực tế đang chạy
- `steering/tech-stack.md` — tech stack thực tế (không phải điền tay)
- `docs/API_CONTRACTS.md` — contracts từ code hiện có
- `logs/CONTEXT_SNAPSHOT.md` — trạng thái khởi đầu

Sau khi chạy xong, human chỉ cần **review và điền những phần còn thiếu** thay vì viết từ đầu.

---

## Các bước thực hiện

### BƯỚC 1 — Scan cấu trúc project

```bash
# Tổng quan folder structure
find . -type f \
  -not -path "*/node_modules/*" \
  -not -path "*/.git/*" \
  -not -path "*/dist/*" \
  -not -path "*/.next/*" \
  -not -path "*/build/*" \
  -not -path "*/__pycache__/*" \
  -not -path "*/.venv/*" \
  | head -100

# Detect monorepo structure
ls -la | grep -E "frontend|backend|apps|packages|services"

# Check package files
find . -maxdepth 3 -name "package.json" -not -path "*/node_modules/*"
find . -maxdepth 3 -name "pyproject.toml" -o -name "requirements.txt" -o -name "go.mod"
```

### BƯỚC 2 — Detect tech stack

```bash
# FE Detection
cat frontend/package.json 2>/dev/null | grep -E '"next|react|vue|svelte|angular|expo"'
cat frontend/package.json 2>/dev/null | grep -E '"tailwind|styled|chakra|material"'
cat frontend/package.json 2>/dev/null | grep -E '"zustand|redux|jotai|recoil"'
cat frontend/package.json 2>/dev/null | grep -E '"jest|vitest|playwright|cypress"'

# BE Detection
cat backend/package.json 2>/dev/null | grep -E '"express|fastify|nestjs|hono"'
cat backend/requirements.txt 2>/dev/null | grep -E 'fastapi|django|flask|sqlalchemy'
cat backend/go.mod 2>/dev/null | head -20

# Database Detection
find . -name "*.sql" -not -path "*/node_modules/*" | head -5
find . -name "prisma" -type d
find . -name "alembic" -type d
grep -r "postgresql\|mysql\|mongodb\|sqlite" --include="*.env*" --include="*.example" . 2>/dev/null | head -5

# Infrastructure
cat docker-compose.yml 2>/dev/null | grep -E "image:|services:" | head -20
ls .github/workflows/ 2>/dev/null
```

### BƯỚC 3 — Scan API endpoints (BE)

```bash
# Express/NestJS
grep -rn "@Get\|@Post\|@Put\|@Patch\|@Delete\|router\.\|app\." \
  backend/src/ --include="*.ts" 2>/dev/null | grep -v "test\|spec" | head -30

# FastAPI
grep -rn "@app\.\|@router\." \
  backend/ --include="*.py" 2>/dev/null | grep -v "test" | head -30

# Laravel
grep -rn "Route::" \
  backend/routes/ --include="*.php" 2>/dev/null | head -30
```

### BƯỚC 4 — Scan API calls (FE)

```bash
# Tìm API service layer
find frontend/src -name "*.service.ts" -o -name "*.api.ts" 2>/dev/null | head -10

# Scan actual calls
grep -rn "fetch\|axios\.\|api\." \
  frontend/src/ --include="*.ts" --include="*.tsx" 2>/dev/null \
  | grep -v "test\|spec\|node_modules" | head -30
```

### BƯỚC 5 — Scan DB schema

```bash
# Migration files
find . -name "*.sql" -not -path "*/node_modules/*" | sort | tail -10
find . -path "*/migrations/*.py" | sort | tail -5

# Prisma schema
cat backend/prisma/schema.prisma 2>/dev/null | head -60

# SQLAlchemy models
grep -rn "class.*Base\|__tablename__" backend/ --include="*.py" 2>/dev/null | head -20
```

---

## Rules

- KHÔNG overwrite nếu file đã có nội dung thật — hỏi human trước
- Luôn mark auto-generated content với `> Auto-generated` và `[TODO]`
- Không invent thông tin không detect được từ code — để `[TODO]`
- Nếu không detect được tech stack rõ ràng → hỏi human, không guess
- Chạy từ root folder của project — không chạy từ subfolder

---

## Khi nào KHÔNG chạy

- Greenfield project (chưa có code) → dùng templates trống có sẵn
- Đã chạy rồi và docs đã được human fill → chỉ update phần thay đổi
